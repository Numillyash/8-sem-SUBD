# ЛР3. Первая часть: SQL-запросы

## Стенд

Работа выполнялась в отдельной базе данных `subd_lab3`, развернутой в Docker-контейнере PostgreSQL 16.

Параметры подключения:

- host: 127.0.0.1
- port: 35432
- database: subd_lab3
- user: lab3_admin

## Использованные отношения

Для выполнения первой части ЛР3 использовались отношения из сданной ЛР1 по варианту 8:

- `documents` — документы организации;
- `document_operations` — журнал операций с документами.

Дополнительно были добавлены тестовые записи `DOC-009` и `DOC-010`, чтобы запросы ЛР3 давали непустые и проверяемые результаты.

## Контроль количества данных

По результатам проверки:

- `documents`: 10 записей;
- `document_operations`: 13 записей.

## Результаты запросов

### Запрос 1

Найти ответственных сотрудников, отвечающих более чем за один секретный документ заданной степени секретности.

Параметр:

- степень секретности: `секретно`.

Результат:

- `Сотрудник_А`, 2 документа.

### Запрос 2

Найти документы, срок окончания действия которых не входит в перечень заданных месяцев.

Параметры:

- год: 2025;
- исключаемые месяцы: май, июнь, июль.

Результат:

- `DOC-003`, `Сотрудник_Б`, 2025-03-01;
- `DOC-008`, `Сотрудник_Д`, 2025-08-01;
- `DOC-010`, `Сотрудник_Г`, 2025-10-01.

### Запрос 3

Найти сотрудников, совершивших с каким-либо документом все возможные операции, либо сотрудников, ответственных за документы, но не выполнивших с ними ни одной операции.

Параметр:

- возможные операции: `Создание документа`, `Проверка документа`, `Согласование документа`.

Результат:

- `Сотрудник_А`;
- `Сотрудник_В`;
- `Сотрудник_Г`.

## Основные файлы

- `sql/01_schema.sql` — создание таблиц;
- `sql/02_insert_documents.sql` — добавление документов;
- `sql/03_insert_operations.sql` — добавление операций;
- `sql/04_insert_lab3_extra_data.sql` — дополнительные данные для ЛР3;
- `sql/10_part1_queries.sql` — запросы первой части;
- `sql/11_part1_checks.sql` — проверочные запросы;
- `logs/10_part1_queries.log` — результат выполнения запросов.
